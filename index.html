Quick Read/Background Information before Jumping into Hands on Coding


The World-Wide-Web is a vast collection of documents that can be accessed from anywhere, at any time, by anyone in possession of the correct address (url). These documents, the majority of which are coded in HTML, are connected to each other by hyperlinks.

Hyper Text Markup Language (HTML) is a collection of code that defines what the users of web documents will view on web pages. Steve Huffman explains that HTML is the glue that holds the web together and hyperlinks are the means of transportation between them.


HTML defines what the viewer:
  • is able to see
  • what the documents (pages) will look like
  • and references (links) to other documents


Initial HTML vocabulary includes:
  • Tag—a letter or group of letters that are surrounded by these two brackets < > (this is an opening tag), or preceded by a forward slash in these two brackets </ > (this is a closing tag)
  • Element—the content that goes between the opening and closing tags above, this will be displayed as the tags dictate
  • Attribute—describes a property of the html element in <a href=”http://www.urbandictionary.com” > the information following "href=" will provide the computer processor with the information it needs in this case to link to Urban Dictionary



Warning

Some claim that computers are stupid. Computers can’t really be stupid since they are not sentient; they operate in a binary system of electrical circuits and can only make a distinction between open or closed. A coder may also think of open or closed as yes or no.

The coder uses HTML to instruct the computer processor to create a change such as bold or italic to specific elements in the text, they must also tell the computer when to stop taking this action.

The computer’s circuits do not have the ability to ask themselves if the programmer really intended to use bold for the entire last half of the document, the programmer must anticipate this issue.



HTML is composed of tags which fall into three categories:

  • Mark Up—which make changes such as bold or italics
  • Anchor—code that provides directions for a web address thereby creating a link
  • Image—inserts images into web pages along with its attributes. The attributes include where the item will be found (url) along with a description or text explanation of what the item is if the link can’t be viewed. This is important for two reasons, the first is the user may be able to find the link on their own and more importantly it addresses the requirements of Universal Design, which ask the programmer to anticipate the needs of visually impaired web cruisers



Creating White Space

White Space is an essential element for making webpages easy on the viewer. When content is displayed with gaps for the eye to rest on the viewer can interact more efficiently in the non-print based medium of the World-Wide-Web.

Web pages are more likely to be read, enjoyed and revisited if the viewer is not slammed by a wall of words. Web cruisers are not going to give a page any more attention than they have too and white space helps them scan for content that is interesting to them.


Let’s Do a Little Coding

First: MarkUp Bold and Italics

For bold the HTML code looks like: <b>I am bold</b>
The viewer sees I am bold

For Italics (emphasis) the code looks like: <em>I am italic</em>
The viewer sees I am italic

Recall that the open commands are <b> and <em> and the required closing command needs a forward slash, </b> and </em>. The processor will not stop doing any instruction you start until you tell it to stop doing so.



Second: Anchor Providing the processor with a web address/url

The code looks like: <b> a href=www.linkyouwant.com>clickable name</a>
The viewer sees a clickable link


Third: Inserting an Image

The code looks like: <img scr=”url” alt=”text description”>
 The viewer sees the picture (jpeg that you want seen)

Notice that there is no </img> this is because inserting pictures falls into the category of void tags, ones that simply do one thing, (insert an image or a white space) and then return to original coding structure.


Fourth: Two Methods for Creating White Space

<br> and <p>, line break and paragraph.

  • <br> is a void tag (does not need a closing tag) along with being an inline tag
  • <p>block</p> is a block tag which creates a container for larger amounts of content. It will stretch to contain anything placed within the opening and closing commands while providing a border of whitespace around the content


Fifth: Container Elements

<span> and <div>, inline and block

  • <span class=” “> text </span> is an inline container
  • <div class=” “ > text </div> is a block container


Concluding Notes for Coders While Rounding Up Information

  • Understand the when the programmer starts something they must also stop it
  • Void commands do not require a stop command
  • Commands may be inline or block, spaces or containers
  • Without HTML the viewer will encounter a wall of words and jump over your site


There are some elements in HTML code that do not have to be closed after being opened, these are void elements. Two that are learned early on are break <br> and image <img scr=”url” alt=”text”>. With <br> a blank space is added and the action is done. This is also true for <img…..> which inserts a picture and once done there is no further action to take.
